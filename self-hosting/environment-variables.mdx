This page documents all environment variables available in Chatwoot. These variables are used to configure various aspects of your self-hosted installation.

## Core Configuration

<ParamField path="SECRET_KEY_BASE" type="string" required>
  Used to verify the integrity of signed cookies. Must be alphanumeric without special characters or symbols.
  
  Generate using: `rake secret`
  
  **Security:** Ensure this is a lengthy, secure, random value unique to your installation.
</ParamField>

<ParamField path="FRONTEND_URL" type="string" required>
  The URL where your Chatwoot application will be accessible.
  
  **Example:** `https://chatwoot.example.com`
  
  **Default:** `http://0.0.0.0:3000`
</ParamField>

<ParamField path="HELPCENTER_URL" type="string">
  Dedicated URL for help center pages if you want to host them on a separate domain.
  
  **Example:** `https://help.example.com`
</ParamField>

<ParamField path="RAILS_ENV" type="string">
  Rails environment mode.
  
  **Options:** `development`, `production`, `staging`
  
  **Default:** `development`
</ParamField>

<ParamField path="RAILS_MAX_THREADS" type="number">
  Maximum number of threads for the Rails application server.
  
  **Default:** `5`
</ParamField>

## Security & Encryption

<ParamField path="FORCE_SSL" type="boolean">
  Force all access to the app over SSL and use secure cookies.
  
  **Default:** `false`
  
  **Recommendation:** Set to `true` in production environments.
</ParamField>

<ParamField path="ACTIVE_RECORD_ENCRYPTION_PRIMARY_KEY" type="string">
  Primary encryption key for Active Record Encryption. Required for MFA/2FA functionality.
  
  Generate using: `rails db:encryption:init`
  
  **Important:** Use different keys for each environment (development, staging, production).
</ParamField>

<ParamField path="ACTIVE_RECORD_ENCRYPTION_DETERMINISTIC_KEY" type="string">
  Deterministic encryption key for Active Record Encryption. Required for MFA/2FA functionality.
  
  Generate using: `rails db:encryption:init`
</ParamField>

<ParamField path="ACTIVE_RECORD_ENCRYPTION_KEY_DERIVATION_SALT" type="string">
  Key derivation salt for Active Record Encryption. Required for MFA/2FA functionality.
  
  Generate using: `rails db:encryption:init`
</ParamField>

## Account & Signup Management

<ParamField path="ENABLE_ACCOUNT_SIGNUP" type="string">
  Controls new sign ups on your Chatwoot installation.
  
  **Options:**
  - `true` - Default option, allows sign ups
  - `false` - Disables all endpoints related to sign ups
  - `api_only` - Disables UI for signup, but allows sign ups via account APIs
  
  **Default:** `false`
</ParamField>

<ParamField path="DEFAULT_LOCALE" type="string">
  Default locale for the application.
  
  If set, all non-authenticated pages will fallback to this locale. When a new account is created, this will be the default language.
  
  **Default:** `en`
</ParamField>

## Asset & CDN Configuration

<ParamField path="ASSET_CDN_HOST" type="string">
  CDN host URL for serving static assets.
  
  **Example:** `https://cdn.example.com`
</ParamField>

<ParamField path="RAILS_SERVE_STATIC_FILES" type="boolean">
  Enable serving static files from the `/public` folder.
  
  **Default:** `true`
  
  **Note:** In production, it's recommended to use NGINX or Apache to serve static files.
</ParamField>

## Logging Configuration

<ParamField path="RAILS_LOG_TO_STDOUT" type="boolean">
  Write logs to stdout instead of log files.
  
  **Default:** `true`
  
  **Note:** Useful for containerized deployments and cloud platforms.
</ParamField>

<ParamField path="LOG_LEVEL" type="string">
  Application log level.
  
  **Options:** `debug`, `info`, `warn`, `error`, `fatal`
  
  **Default:** `info`
</ParamField>

<ParamField path="LOG_SIZE" type="number">
  Maximum log file size in megabytes (when not logging to stdout).
  
  **Default:** `500`
</ParamField>

<ParamField path="LOGRAGE_ENABLED" type="boolean">
  Use lograge for structured logging instead of default Rails logger.
  
  **Default:** `false`
</ParamField>

## Performance & Rate Limiting

### Rack Attack

<ParamField path="ENABLE_RACK_ATTACK" type="boolean">
  Enable Rack Attack for rate limiting and abuse prevention.
  
  **Default:** `false`
</ParamField>

<ParamField path="RACK_ATTACK_LIMIT" type="number">
  Maximum number of requests per period.
  
  **Default:** `300`
</ParamField>

<ParamField path="ENABLE_RACK_ATTACK_WIDGET_API" type="boolean">
  Enable rate limiting for widget API endpoints.
  
  **Default:** `false`
</ParamField>

<ParamField path="RACK_ATTACK_ALLOWED_IPS" type="string">
  Comma-separated list of trusted IPs that bypass Rack Attack throttling rules.
  
  **Example:** `127.0.0.1,::1,192.168.0.10`
</ParamField>

### Sidekiq

<ParamField path="SIDEKIQ_CONCURRENCY" type="number">
  Number of concurrent Sidekiq worker threads.
  
  **Default:** `10`
</ParamField>

<ParamField path="ENABLE_SIDEKIQ_DEQUEUE_LOGGER" type="boolean">
  Enable verbose logging each time a job is dequeued in Sidekiq.
  
  **Default:** `false`
</ParamField>

### Database Performance

<ParamField path="DB_POOL_REAPING_FREQUENCY" type="number">
  Frequency in seconds to periodically run the connection pool reaper.
  
  **Default:** `30`
</ParamField>

### Housekeeping

<ParamField path="REMOVE_STALE_CONTACT_INBOX_JOB_STATUS" type="boolean">
  Remove stale contact inboxes with no conversation older than 90 days.
  
  **Default:** `false`
</ParamField>

## Advanced Configuration

<ParamField path="CW_API_ONLY_SERVER" type="boolean">
  Run Chatwoot as an API-only server, disabling frontend dashboard endpoints.
  
  **Default:** `false`
</ParamField>

<ParamField path="DIRECT_UPLOADS_ENABLED" type="boolean">
  Enable direct uploads to cloud storage using signed URLs.
  
  **Note:** Requires proper CORS configuration on your cloud storage. See [Active Storage CORS](https://edgeguides.rubyonrails.org/active_storage_overview.html#cross-origin-resource-sharing-cors-configuration)
  
  **Default:** `false`
</ParamField>
